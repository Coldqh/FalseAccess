# FALSE ACCESS 0.8.1 — GREYLOCK-09

## Новая техническая глава

- полноценный активный инцидент в инфраструктуре Greylock;
- SIEM и EDR работают вместе с IdP, VPN, Windows и Linux telemetry;
- подтверждение алерта, построение process tree и поиск индикаторов по всему контуру;
- определение scope без объявления всей сети скомпрометированной;
- точечный containment с сохранением доказательств и непрерывности расчётного сервиса;
- отключение сервисной учётки и отзыв дочерних сессий;
- удаление Windows scheduled task и Linux systemd timer;
- закрытие слабой VPN-политики;
- staged recovery из проверенного образа;
- health checks, ограниченный трафик и усиленный мониторинг;
- вторая волна через забытый CI PAT в истории репозитория;
- самостоятельное расследование второй волны без готового порядка команд;
- итоговый технический и управленческий разбор;
- скрытые экзамены `siem-edr-incident`, `incident-response` и `stage-4-capstone`.

## Симулятор и прогресс

- временное приложение `GREYLOCK-09` появляется после `DEADFRAME-08` и исчезает после завершения;
- отдельный прогресс главы сохраняется между запусками;
- Incident Response добавлен как отдельный тип процедурных контрактов;
- новые контракты на containment и recovery;
- награды по Incident Response, SIEM, SOC, Security Engineering, DFIR, Threat Hunting, планированию и коммуникации;
- схема сохранений обновлена до `false-access-progress-v18`;
- сохранения 0.8.0 мигрируют автоматически.

## Техническое

- версия приложения, manifest, update manager и PWA-cache обновлены до `0.8.1`;
- мобильные исправления обновлений, баланса и Mail из 0.8.0 сохранены;
- production build: 1643 modules transformed.
