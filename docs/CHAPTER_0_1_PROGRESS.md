# FALSE ACCESS — Chapter 0.1 implementation

## Что заменено

- новый игрок больше не попадает сразу на свободный Desktop;
- Boot ведёт в обязательную сюжетную главу `workspace-01`;
- старая навигационная часть CLINIC-01 заменена новым mission runtime;
- после главы старые цели `pwd`, `ls`, `cd-case`, `read-brief` мигрируются как уже доказанные действия;
- следующая стадия CLINIC-01 начинается с журналов и процессов;
- Terminal больше не вставляет готовые команды для текущей стадии;
- действия Terminal записываются в evidence journal.

## Учебный цикл

1. входящий локальный пакет Максима;
2. короткая модель cwd и пути;
3. guided navigation к `brief.txt`;
4. самостоятельный поиск неизвестного вложенного каталога;
5. открытие артефакта;
6. evidence link;
7. semantic assessment;
8. дебриф;
9. переход к главе 0.2.

## Два допустимых пути

- последовательная навигация относительными путями;
- прямое открытие артефакта абсолютным путём.

## Безопасность

- файловая система синтетическая;
- изменение данных блокируется;
- внешняя сеть блокируется;
- destructive и external-network attempts фиксируются как critical errors;
- seed детерминирован;
- глава поддерживает reset.

## Автоматические проверки

- детерминированная генерация;
- нормализация путей;
- stdout/stderr/exit code;
- relative и absolute solution family;
- отсутствие прохождения по памяти;
- critical errors;
- evidence-based completion.
