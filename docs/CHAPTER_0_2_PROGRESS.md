# FALSE ACCESS — Chapter 0.2 implementation

## Что добавлено

- обязательная глава `logs-01` между рабочим местом и основной CLINIC-01;
- новый shell runtime для главы с pipelines, stdout, stderr и exit codes;
- безопасное перенаправление производных данных только в `/home/ilya/work`;
- read-only защита исходных файлов;
- блокировка внешней сети в любой части pipeline;
- команды `grep`, `wc`, `cut`, `sort`, `uniq`, `find`, `jq`, `date`, `head`, `tail`, `file`;
- guided-разбор `auth.log`;
- нормализация UTC+03:00 в UTC и сопоставление с `proxy.csv`;
- independent transfer на сгенерированном JSONL-наборе;
- повреждённая строка, меняющийся источник, счётчик и имя файла;
- три evidence links вместо свободного поля ответа;
- повторный запуск на новом seed после прохождения;
- обновлённый маршрут Missions и переход к главе 0.3.

## Допустимые пути transfer

1. структурированный pipeline через `jq`;
2. текстовый pipeline через `cut`.

Оба пути должны получить правильный источник и количество failed-событий из реально выполненных команд.

## Не считается доказательством

- введённый вручную IP;
- вручную посчитанное число;
- открытие подсказки;
- просмотр файла без обработки;
- наличие правильных слов в команде без правильного результата.

## Безопасность

- внешняя сеть отключена;
- исходные артефакты read-only;
- запись разрешена только в рабочую производную директорию;
- destructive и network attempts становятся critical errors;
- генерация детерминирована seed;
- миссия поддерживает reset и reroll.

## Проверки

- TypeScript-проверка доменного слоя;
- проверка валидности MissionDefinition;
- smoke-прохождение полного jq-маршрута;
- проверка альтернативного cut-маршрута;
- проверка stderr и exit codes;
- проверка read-only boundary;
- проверка блокировки внешней команды внутри pipeline;
- тесты детерминированной генерации и semantic completion.
