# FALSE ACCESS — MVP 0.2

Рабочий вертикальный срез сюжетной RPG и локального тренажёра по информационной безопасности.

## Что уже работает

### Сюжетная глава

- загрузочный экран и виртуальная ОС;
- адаптивный ПК/телефон;
- окна, панель задач и локальные уведомления;
- первая миссия `CLINIC-01`;
- терминал на xterm.js с виртуальной файловой системой;
- команды `pwd`, `ls`, `cd`, `cat`, `grep`, `ps`, `ss`, `tree`;
- настоящий Python в браузере через Pyodide;
- объяснение переменных, циклов, условий, отступов и f-строк;
- SIEM с первым triage;
- технический отчёт с проверкой ключевых фактов;
- почта, мессенджер, навыки и локальное сохранение;
- финал главы с предложением первой работы.

### Повторяемый симулятор

- приложение `Work Queue`;
- три работодателя с отдельной репутацией;
- локальная экономика и история заказов;
- процедурная сборка задач из безопасных шаблонов;
- задания по Linux, сетям, Python, SOC и Web Security;
- меняющиеся IP-адреса, процессы, журналы, домены и ответы;
- отдельный shell для каждого заказа;
- реальный Python-код в заданиях через WebAssembly;
- режим чистого выполнения без подсказки;
- оплата и репутация за подтверждённый результат.

### Сайты мира

Встроенный браузер открывает реальные статические страницы, которые также можно открыть отдельной вкладкой:

- `sites/sfera/`;
- `sites/ostrogorsk-news/`;
- `sites/clinic/`.

Все цели, адреса и данные вымышлены. Лаборатории не обращаются к внешним системам.

## Запуск

```bash
npm ci
npm run dev
```

## Проверка production-сборки

```bash
npm run build
npm run preview
```

## GitHub Pages

1. Открыть **Settings → Pages**.
2. В **Source** выбрать **GitHub Actions**.
3. Workflow `.github/workflows/deploy.yml` соберёт и опубликует игру.

## Порядок сюжетного прохождения

1. Пройти вводную.
2. Открыть `Missions`.
3. Выполнить команды в `Terminal`.
4. Дописать Python-скрипт в `Code Editor`.
5. Разобрать алерт в `SIEM`.
6. Сдать отчёт в `Notes`.
7. Прочитать предложение о работе в `Mail`.
8. Вернуться в `Work Queue` и брать повторяемые заказы.
